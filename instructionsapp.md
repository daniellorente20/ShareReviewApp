# Arbolus — ReviewShare Live Coding Interview

Welcome! 👋

In this live coding session, you will build a simplified **Software Review Hub** called **ReviewShare**.

The goal of this exercise is **not to finish everything**, but to help us understand how you:

* Think through problems
* Make technical decisions
* Prioritize features
* Use AI tools effectively
* Communicate your reasoning

Please **think out loud throughout the session**.

---

## ⏱️ Timebox

You have **2 hours** for this exercise.

We recommend:

* Spend a few minutes planning before coding
* Commit your work periodically
* Focus on delivering a working MVP first

---

## 🧩 The Challenge

Build a web application where users can:

* Browse software products
* Leave reviews
* Explore a feed of reviews

You may use **any tech stack** and **any AI tools** (Copilot, Cursor, Claude, ChatGPT, etc.).

---

## ✅ MVP Requirements (Required)

Please prioritize these features.

---

### 1. Software Catalog

Create a seeded list of **5–10 software products**.

Each product should include:

* name
* category
* description

The data can be hardcoded or stored in a database.

---

### 2. Leave a Review

Authenticated users should be able to:

* Rate software (1–5 stars)
* Write a text review

> Note: Authentication can be simplified. A mock or basic implementation is acceptable.

---

### 3. Review Feed

Users should be able to browse reviews with basic filtering by:

* software
* category

Focus on functionality over visual polish.

---

## ⭐ Stretch Goals (Optional)

Only attempt these if the MVP is working:

* Search and sorting
* Review helpfulness voting
* User profiles
* Analytics dashboard
* Responsive design improvements
